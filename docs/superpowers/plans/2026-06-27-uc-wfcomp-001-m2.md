## Design-Entscheidung (vor Bau bestätigen)

**Nur b (Zweck) + c (Betroffenenkategorie) bekommen LLM-Inferenz.** a (Verantwortlicher = Org-Metadatum), f (Löschfrist), g (TOM-Adäquanz) bleiben **reines Ask** (keine LLM-Vermutung — sie stehen nicht in der Workflow-Struktur und sind Policy-Entscheidungen). YAGNI + Honesty: nicht raten, wo keine strukturelle Evidenz ist.

**Privacy-Verstärkung:** Der LLM-Prompt bekommt NUR die sanitisierte Struktur (Node-Typen/Namen, Feld-**Keys**, Domains) — niemals Werte/pinData. Das ist .0 (THE-358) im LLM-Pfad fortgesetzt.

**G7-Invariante:** Ein LLM-Vorschlag macht ein Feld NIE `present`. Status bleibt `needs_attestation` (+ `mode:'confirm'` + `suggestion`). Nur `applyAttestation` (Mensch) flippt auf `present`.

## Task M2.1 (.3 / THE-354): `inference.ts` — LLM-Vorschläge (G6/G7-Input)

**Files:** Create `src/services/wfcomp/inference.ts` + `src/prompts/wfcompInference.prompt.ts` · Test `src/__tests__/wfcomp-inference.test.ts`

- `inferLegalFields(sanitized, opts?: { anthropicClient?: Anthropic }): Promise<FieldSuggestion[]>`
  - Baut Prompt aus sanitisierter Struktur. Fragt NUR nach b + c.
  - Antwort Zod-validiert: `{ suggestions: [{ litera:'b'|'c', value, confidence(0..1), rationale }] }`.
  - **Abstain-Regel:** confidence < THRESHOLD (0.5) → Vorschlag verworfen → Feld bleibt `ask`. Kein erfundener Zweck bei mehrdeutigem Workflow.
  - `FieldSuggestion = { litera, value, confidence, rationale, provenance:'ai_generated' }`.
- TDD (mock-client wie `makeMockAnthropic`):
  - `inferrable-purpose` → liefert b-Vorschlag, confidence ≥ 0.5.
  - `ambiguous-purpose` (mock LLM: low confidence) → **kein** Vorschlag (abstain). **= G6 (keine Halluzination bei Unklarheit).**

## Task M2.2 (.5-Mechanik / THE-356): `attestation.ts` — Ask/Confirm + Recompute

**Files:** Create `src/services/wfcomp/attestation.ts`; extend `types.ts` (`FieldResult.mode?`, `.suggestion?`) · Test `src/__tests__/wfcomp-attestation.test.ts`

- `annotateModes(report, suggestions): GapReport` — jedes `needs_attestation`-Feld bekommt `mode:'confirm'` (wenn Suggestion vorhanden) sonst `mode:'ask'`; Suggestion wird angehängt. **G8.**
- `applyAttestation(lifted, attestations): LiftedGraph` — bestätigte Antwort als gelifteten Knoten injizieren (z.B. b bestätigt → `goal{Purpose}` + `realization`-Edge; c bestätigt → `business_object{DataSubjectCategory}` + `association`). Danach `runTraceCheck` → Feld flippt auf `present`. **G9 (Recompute).**
- `assessWorkflowWithInference(raw, opts): Promise<GapReport>` = sanitize → scope → lift → inferLegalFields → trace → annotateModes.
- TDD:
  - `inferrable-purpose` nach Inferenz: b `mode:'confirm'` + suggestion; a `mode:'ask'`. **G8.**
  - `applyAttestation(b)` → recompute → b `present`. **G9.**
  - Über alle Fixtures: nach Inferenz ist KEIN Rechtsfeld `present`. **G7.**
