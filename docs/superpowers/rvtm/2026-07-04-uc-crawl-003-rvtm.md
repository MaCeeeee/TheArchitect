## Driver (warum)

The Architect erzeugt bei Skalierung **viele Mandanten × viele Architektur-Anfragen → sehr viele Scrapes** (Korpus-Feeding, RADAR, AUTOCRAWL, Regulation-Refresh). Firecrawl-**Cloud** rechnet **pro Call** ab → Skalierungs-Kostentreiber. Server B hat bereits versenkte Fixkapazität. Entscheidung (2026-07-03): Scraping-Backend Cloud → **self-hosted Firecrawl auf Server B**, mit **Cloud als Fallback**.

## Root Cause des Ist-Ausfalls

Vendor hat die Architektur umgebaut: `api` hängt jetzt zwingend an **RabbitMQ** + **nuq-postgres** (via `NUQ_RABBITMQ_URL`), plus `playwright-service` + `redis`. Die eingefrorene April-Config (Redis/BullMQ + Supabase-Ära) liefert diese nicht → `docker compose up` bricht vor Container-Erstellung ab. Verstärker: Coolify deployt `git_branch: main` (floating) → genau dieser Bruch. Prod läuft aktuell fehlerfrei über **Cloud** (325 Artikel, 0 Fehler).
