## Glossary

* **AC** — Acceptance Criterion (nummeriert pro REQ)
* **REQ** — Sub-requirement (Linear-Issues **noch anzulegen** nach Plan-Approval, hier als REQ-PROV-001.x referenziert)
* **Producer** — Codepfad, der ein Element/eine Connection erzeugt oder updated
* **Atom** — ein ArchiMate-Element ODER eine `CONNECTS_TO`-Connection
* **Verification:** `Test` = Unit/Integration · `Inspection` = Code-Review/Static · `Demo` = manueller Walk · `Analysis` = Metrik/Messung

## Naming-Entscheidung (Foundation — gilt für alle REQs)

| Feld | Status | Regel |
|---|---|---|
| `provenance: 'user'\|'import'\|'ai_generated'\|'mcp_discovered'` | **NEU** | additiv, null Konflikt |
| `certifiedBy?: string` · `certifiedAt?: string` | **NEU** | null Konflikt |
| `source?: string` | **ADOPTIEREN** (existiert) | Werte standardisieren, nie blind überschreiben |
| `confidence?: number` (0–1) | **ADOPTIEREN** (existiert) | nur setzen wo real messbar |

`deriveProvenance(source)`: `'ai-heal'`/`'compliance-requirement'`→`ai_generated` · `'csv'`/`'bpmn'`/`'n8n'`/`'blueprint'`→`import` · sonst→`user`.

## Risk Register

| Risk | Mitigation |
|---|---|
| Vergessener Producer schreibt stumm keine Provenance | REQ-2-AC1 Inventar + shared Helper + grep-Static-Check (Cross-Cutting) |
| Backfill überschreibt Auto-Heal-`source`/`confidence` | REQ-4-AC2 scoped `WHERE provenance IS NULL` + byte-identisch-Test |
| Konfidenz-Werte erfunden/unkalibriert | `confidence` nur wo real (LLM-Signal / Match-Score-Reuse) — nie Default-Wert |
| Zod droppt Feld lautlos bei Teil-Rollout | REQ-3-AC2 Schemas zuerst, im selben PR wie Write-Pfade |
| Mongo- vs. Neo4j-Divergenz | ArchiMate-Atome leben **nur** in Neo4j (kein Mongo-Element-Model) → Single-Store für Provenance; Mongo betrifft nur `metadataJson`-Altlast |
