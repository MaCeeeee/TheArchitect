## Ziel

Die Coverage-Decke verschieben: **Coverage = Korpus statt Coverage = 7 Regeln.** Jedes gecrawlte
Gesetz ist automatisch discovery-fähig, ohne Regel-Zeile. Verbleibende Decke = Korpus-Inhalt
(dessen Wachstum ist die andere Achse: UC-ICM-001/UC-AUTOCRAWL, THE-457 AC-3) — ehrlich
ausgewiesen nach dem F1-Muster (THE-455).

## Abgrenzung zu UC-LAW-001 (THE-454)

| | UC-LAW-001 (Stage A, gebaut) | UC-LAW-002 (Stage B, dieses UC) |
|---|---|---|
| Mechanik | 7 kuratierte Regeln, deterministisch | Embedding-Retrieval + LLM-Judge über den GESAMTEN Korpus |
| Kosten | 0 (kein LLM) | LLM-Calls je Discover-Lauf (expliziter Trigger, Cache) |
| Decke | Regel-Tabelle | Korpus-Inhalt |
| Reproduzierbar | vollständig | Stufen .1/.2 deterministisch; Judge gemessen statt behauptet (.6) |
| Verdikt-Charakter | autoritativ | Vorschlag (`auto`), Mensch bestätigt |

## Lösungsskizze

**Stage B** (expliziter Nutzer-Trigger im ApplicabilityCheck-Panel):

1. **Profil (.1, THE-460):** deterministische Modell-Verdichtung — Projekt-Kontext + Element-Summaries
   je Layer + LAW-001-Signale als Hints; budgetiert, priorisiert (PII/ai_agent/Criticality zuerst).
2. **Retrieval (.2, THE-461):** Query-Embedding (gleiche Infra wie Sprint-2 Track A, gleiches Modell
   wie Korpus: all-mpnet-base-v2 768-dim) → Top-K Paragraphen über ALLE Sources → Aggregation zu
   Kandidaten-Gesetzen (Score je Source, Sprachfamilien-Merge).
3. **Judge (.3, THE-462):** LLM je Kandidat — applies/confidence/reasoning/elementIds/keyParagraphs;
   UC-ICM-002-Bausteine (validateConfidence, Provider-Fallback, AiTrace); Schwellen-Gating + Cache
   `(profileHash, source, corpusVersionHash)`; geschlossene Kandidatenmenge (Anti-Halluzination).
4. **Merge (.4, THE-463):** Provenance `rules | corpus | both`; deterministische Urteile nie
   überschrieben; Korpus-Befunde mit Lifecycle `auto → confirmed | rejected` persistiert (Dedup bei
   Re-Runs); LAW-001 bleibt ohne Keys/Korpus unverändert (Regression).
5. **UI (.5, THE-464):** „Discover from corpus"-Button (Kosten-Hinweis, Key-/Korpus-Gating),
   Provenance-Badges, Confirm/Reject, Paragraph+Element-Evidenz, Add-to-pipeline (THE-390 P4b),
   Deckungs-Zeile „Stage A: N Regeln · Stage B: M Korpus-Gesetze".
6. **Evals (.6, THE-465):** Golden-Set ≥10 Architektur-Fixtures (inkl. erwarteter Nicht-Treffer),
   Precision/Recall je Gesetz + Confidence-Kalibrierung + FP/FN-Bias (THE-430-Philosophie),
   Determinismus-Tests für .1/.2, Kosten-Regression, DE/EN-Familien-Konsistenz, Fixture-Korpus.

## Grenzen / Nicht-Ziele

- LLM nur in Stage B; ohne API-Keys degradiert sauber auf UC-LAW-001.
- Kein Auto-Confirm (Trust-Spine); kein Crawling neuer Gesetze in diesem UC; kein Freitext-Input in v1.
- Deckung = Korpus, nicht Rechtsordnung — im Report ausgewiesen.
