### REQ-RED-004 Bulk-Merge mit Per-Pair-Decisions (commit `7f2715a`)

**Strategischer Pivot:** statt Cross-Type zuerst (Plan) → Bulk-Merge zuerst gewählt. Grund: UC-RED-001 war read-only, User konnte sehen aber nicht handeln. Bulk-Merge macht aus Tool ein Action-Tool.

- **Neuer Service** `redundancyResolution.service.ts` (208 LoC, +12 unit-tests):
  - `mergeElements(projectId, sourceId, targetId)` mit 5-Schritte-Sequenz (existence-check → outgoing-transfer → incoming-transfer → DETACH DELETE → deleteEmbedding fire-and-forget)
  - `applyRedundancyDecisions(projectId, decisions[])` mit per-decision-independence (one failure blockt nicht den Rest)
  - 4 Decision-Actions: `merge-into-a` / `merge-into-b` / `keep-both` / `skip`
- **Neue Route** `POST /:projectId/redundancies/resolve` mit Zod-validation, batch max 50, riskLevel high, PERMISSIONS.ELEMENT_DELETE, +10 supertests
- **Frontend** RedundancyPanel komplett re-modelliert:
  - Per-Pair Decision-Buttons mit visuellem Feedback (grüner Ring um Target)
  - Footer-Counter "X merge · Y keep · Z skip"
  - Apply-Button mit Count im Label
  - Nach erfolgreichem Apply: scene-reload + pair-list-refresh
  - Toast mit result-counts

**Production-Validation:** Toast "✓ 1 merged" im UI, Element-Counter von 61→60, Pair-Counter von 20→17 (3 Pairs verschwunden weil das gemergte Element in 3 Beziehungen war).

### REQ-RED-002 Cross-Type via Semantic Groups (commit `3da2032`)

### REQ-RED-005 Audit + Stats-Endpoint (commit `53646e4`)

- **Service** schreibt pro Decision Audit-Entry:
  - `redundancy_resolved` (medium risk) für Merges mit `{aId, bId, action, sourceId, targetId}`
  - `redundancy_kept` (low risk) für Keep-both
  - Kein Audit für Skip (pure metadata, no state change)
  - Audit-Failure ist fire-and-forget, blockt nicht den Merge-Result
- **Neuer Endpoint** `GET /:projectId/stats/redundancies` aggregiert via AuditLog.countDocuments + findOne:
  - `totalResolved`, `totalKept`, `lastResolvedAt`, `lastResolvedBy`, `lastResolvedPair`
  - PERMISSIONS.ELEMENT_READ (read-only)
- **+8 Tests** (5 service + 3 route)
