Plus eine ergänzende Klärungsfrage: **"Sensitivity vs Redundancy — warum heißt es so?"** → erklärt: orthogonale Konzepte. Sensitivity = wie schutzbedürftig (B1), Redundancy = gibt es Duplikate (B2). Heute haben wir beide gebaut. Term "Sensitivity" ist Compliance-Domain-Vocabulary (DSGVO Art. 30, ISO 27001 Annex A.8).

## Deploy-Pitfalls die in Erinnerung bleiben sollen

Zwei Mal heute beim Deploy gestolpert, beide klar identifiziert:

1. **rsync vergessen vor Build (Vormittag-Deploy):** `docker compose up -d app` mit altem Image weil Build nicht lief. Fix: `docker exec <container> grep -c "<neuer-string>"` auf laufendem Container — muss >0 sein, sonst neuer Code nicht aktiv.
2. **Hostinger-Web-Terminal vs Mac-Terminal-Verwechslung:** rsync von VPS-internem Terminal zu sich selbst = SSH-Loopback ohne Key-Auth → scheitert. rsync muss vom Mac-Terminal kommen.
